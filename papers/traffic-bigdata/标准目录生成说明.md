# 标准目录生成说明

## 1. 目标

本项目生成的目录不是手动输入的文字目录，而是接近 Word/WPS“引用-目录-更新目录”的标准自动目录。这样处理后，后续论文内容增删、页码变化或章节标题调整时，可以在 Word/WPS 中继续通过“更新域/更新目录”刷新目录，不需要人工逐行修改目录页码。

## 2. 总体流程

标准目录的生成分为四步：

1. 先把学校论文或示例论文清理成模板，保留封面、摘要、目录页、页眉页脚、分节符、页码规则和标题样式。
2. 在目录位置插入 Word 的 TOC 域，而不是写死目录文本。
3. 正文生成时，一级、二级、三级标题分别使用 Word 可识别的标题样式或 outline level。
4. 生成 DOCX 后，再调用 LibreOffice headless 打开文档并更新目录域，使目录显示真实标题和真实页码。

## 3. 目录域的原理

Word 自动目录本质上是一个域代码，项目中使用的是：

```text
TOC \o "1-3" \h \z \u
```

含义如下：

| 参数 | 含义 |
| --- | --- |
| `TOC` | 生成目录 |
| `\o "1-3"` | 抓取 1 到 3 级标题 |
| `\h` | 目录项带超链接 |
| `\z` | Web/预览模式下隐藏制表符和页码引导符 |
| `\u` | 允许根据段落大纲级别生成目录 |

这个域放进 DOCX 后，Word/WPS/LibreOffice 会根据标题样式和分页结果自动生成目录文字、点线和页码。

## 4. 标题为什么必须用样式

目录能不能自动更新，关键取决于正文标题是否被 Word 识别为标题层级。

本项目在生成正文时使用 `add_heading()` 函数处理标题：

| 论文标题层级 | Word 样式 | outline level |
| --- | --- | --- |
| 一级标题，如“第1章 绪论” | `Heading 1` | 0 |
| 二级标题，如“1.1 研究背景” | `Heading 2` | 1 |
| 三级标题，如“1.1.1 技术路线” | `Heading 3` | 2 |

如果模板中缺少某个内置 Heading 样式，脚本会退回到普通段落，并补写 `w:outlineLvl`。这样即使样式名不完全一致，Word/WPS 仍然可以把它识别为目录层级。

## 5. 模板阶段做了什么

模板制作阶段主要处理这些内容：

1. 清理原论文中的旧目录条目，只保留目录标题和目录位置。
2. 删除示例论文里的旧正文、旧参考文献、无效提示文字和多余空段。
3. 保留原模板的封面布局、摘要页、目录页、页眉页脚和分节符。
4. 在目录标题后插入 TOC 域占位。
5. 写入 `w:updateFields` 设置，提示 Word/WPS 打开时可以更新域。

核心思路是：模板只负责版式和占位，真实论文内容通过 JSON 和构建脚本填入。

## 6. 构建阶段做了什么

构建阶段主要由当前论文目录中的 `build.py` 完成：

1. 读取 `meta.json`、章节 JSON、参考文献 JSON 和图片资源。
2. 把正文内容写入模板中的正文占位位置。
3. 每个章节标题用标题样式写入，并设置一级标题新开页。
4. 图、表、正文段落按模板格式写入。
5. 调用 `patch_update_fields()` 给 DOCX 写入自动更新域设置。
6. 调用 `fix_headers_footers()` 修复页眉页脚和页码分节。

也就是说，目录页不是单独拼接出来的，而是正文标题、分节页码和目录域共同工作的结果。

## 7. 为什么还要用 LibreOffice 更新目录

DOCX 文件刚生成时，脚本可以写入目录域，但脚本本身并不知道 Word 最终会怎样分页。比如图片大小、表格跨页、字体替换、页眉页脚都会影响页码。

所以生成后必须让真正的排版引擎打开一次文档，并更新域。项目中使用 LibreOffice headless 执行这个过程：

1. 打开生成好的 DOCX。
2. 更新所有 TOC/页码等域。
3. 保存回 DOCX。
4. 再渲染成 PDF 或 PNG 检查页码和目录是否正确。

这一步相当于程序自动完成了 Word/WPS 里的“右键目录-更新整个目录”。

## 8. 页码分节处理

论文通常不是从封面就开始使用阿拉伯数字页码。本项目按常见毕业论文格式处理：

| 区域 | 页码规则 |
| --- | --- |
| 封面 | 不显示页码 |
| 摘要、目录等前置部分 | 罗马数字，如 I、II |
| 正文开始 | 阿拉伯数字，从 1 开始 |
| 参考文献 | 继续正文页码 |

`fix_headers_footers()` 会重新设置不同 section 的 footer 域：

```text
PAGE \* ROMAN
PAGE
```

同时会确保目录区域使用罗马数字，正文区域从 1 重新开始。这样目录中的页码和正文底部页码才能对上。

## 9. 后续修改时怎么操作

如果后续在 Word/WPS 中继续修改论文，建议按下面方式处理：

1. 改章节标题时，不要只改字体大小，要确认它仍然是对应标题样式。
2. 增删正文、图片、表格后，右键目录选择“更新域”或“更新整个目录”。
3. 如果页码不对，优先检查分节符和页脚是否被误删。
4. 不建议手动改目录里的某一行，因为下一次更新目录会被覆盖。

## 10. 一句话总结

标准目录的关键不是“把目录文字排得像目录”，而是让正文标题、TOC 域、分节页码和排版更新流程连起来。只要这几部分是标准的，Word/WPS 就能继续识别和更新目录。
