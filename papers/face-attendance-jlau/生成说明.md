# 论文生成说明

本目录按“模板 + JSON 内容 + 图片资源 + 构建脚本”的方式生成论文，便于后续继续改内容和重新出 Word。

## 主要文件

- `build.py`：读取学校 Word 模板，保留并填充原模板封面，再生成摘要、目录、正文、参考文献、附录和致谢，并输出最终 `.docx`。
- `write_content.py`：根据项目源码、运行截图和参考文献检索结果生成 `meta.json`、`chapters.json`、`references.json`、`appendix.json` 等内容文件。
- `images/`：模型生成的论文图，包括技术路线图、功能结构图、用例图、部署图、包图、E-R 图和流程图。
- `screenshots/`：从项目实际运行页面截取的前台、后台、考勤、请假等页面图。
- `references_auto.json`、`references_auto_tech.json`：由项目检索程序得到的近五年参考文献来源，构建时去重后写入 `references.json`。

## 目录和页码处理

正文从第 1 页开始重新编号，摘要和目录使用前置页码。章节、参考文献、附录、致谢通过新节分页，但只有正文第一节设置 `w:pgNumType start=1`，后续新节删除页码重置属性，让页码连续继承。

目录页采用 Word 目录的视觉样式：标题、二级标题、点线引导符、右侧页码均由脚本生成。这样避免不同 Word/WPS/LibreOffice 自动更新目录时把分页和样式冲乱。若后续正文增删较多，只需要重新运行 `build.py` 后按渲染页数更新 `add_toc()` 里的页码映射。

## 图片显示处理

正文段落使用固定 20 磅行距；插图段落单独设置为单倍行距。这样 Word/WPS 中图片不会被固定行高裁剪，只显示一部分。数据库表和测试表均由脚本写成真实 Word 表格，并只保留上边框、表头下边框和下边框，形成论文常用三线表，不再把表格转换成图片。

## 运行截图处理

项目先导入 `springboot5s7063d3.sql` 并补充员工、部门、设备、打卡、请假、维保和公告等初始化数据，再启动后端接口。前台和后台 `dist` 目录通过本目录的 `serve_dist_proxy.py` 运行，静态页面访问 `/springboot5s7063d3/` 时会代理到 Spring Boot 后端，避免直接静态预览导致接口 404、页面空白或列表无数据。
